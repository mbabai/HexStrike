# HexStrike Copy Style Guide

This guide standardizes player-facing copy in the rulebook and UI.

## 1. Terminology Casing
- Use lowercase for role terms in running text: `active`, `passive`, `active card`, `passive card`, `active text`, `passive text`.
- Use normal grammar: capitalize these terms when they begin a sentence and in headings/labels.
- Keep gameplay phase names in Title Case when they are section titles (for example, `Simultaneous Play Phase`).

## 2. Color Word Styling
When color words appear in player-facing copy, render the word itself in that color.
- Supported words: `purple`, `green`, `blue`, `red`, `orange`, `yellow`, `gold`.
- Use lowercase color words in prose for consistency.

## 3. Rulebook Paragraph Style
- Prefer direct, present-tense instructions.
- Use short declarative sentences before edge-case notes.
- Put requirement statements before examples.

## 4. Card Selection Language
Use this order when describing a play selection:
1. active card
2. passive card
3. rotation

Always state that active/passive cards must be opposite types (movement + ability).

## 5. Consistency Checks Before Merge
For any rulebook or UI copy change, verify:
- `active`/`passive` term casing is consistent.
- Color words are styled using the shared color-term classes.
- Duplicate terms and conflicting phrasing are removed.
- Grammar/typo pass is complete.
